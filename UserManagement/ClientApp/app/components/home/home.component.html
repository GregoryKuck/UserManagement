User Management Assessement

Seeing as Microsoft already did this for me, built with:

  • ASP.NET Core and C# for cross-platform server-side code
  • Angular and TypeScript for client-side code
  • Webpack for building and bundling client-side resources
  • Bootstrap for layout and styling

Implementation information:

  • Shouldn't require any other setup in order to run, although I did encounter an issue with webpack, which needed a separate npm install
  • NPM requires nodejs
  • Uses a static in memory data source
  • Typically, a pre-built claims-based authentications and authorisations component will be used, so this implements a very rudimentary permissions mechanic
  • Very little from this design would be in a production system
  • Uses .Net Core dependency injection

Other notes:

This solution took significantly longer to implement that the allotted 3 hours, although I am going to blame the HGC Mid-Season Brawl at Dream Hack Summer and the world cup football for it.

Due to only a cursory knowledge of angular.io/typescript implementation, I couldn't quite get the angular.io HTTP component hooked up to the API correctly, so as a workaround I resorted to a more monolith-style design which does not implement REST best-practices in terms of verbs, and URL structures.

Styling etc. are defaults and is missing loading indicators.

Due to time constraints, I have not provided unit tests. Instead, I have provided ways to check outcomes through the UI:

  • User Roles to assign a role to a user
  • User-based Ticker Permissions to view tickers based on user name
  • Role-based Ticker Permissions to view tickers based on roles
